# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

1. **Do not** create a public GitHub issue for the vulnerability.
2. Email us at [security@example.com] with details of the vulnerability.
3. Include steps to reproduce the issue if possible.
4. Allow us reasonable time to respond before disclosing publicly.

## What to Expect

- We will acknowledge receipt of your report within 48 hours.
- We will provide an estimated timeline for addressing the vulnerability.
- We will notify you when the vulnerability is fixed.

## Security Best Practices

When contributing to excelart, please:

- Keep dependencies up to date
- Never commit sensitive data (API keys, passwords, etc.)
- Follow secure coding practices
- Report any security concerns promptly

Thank you for helping keep excelart secure!
